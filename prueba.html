Apuntes de clase: Lenguaje de la web: HTML y XHTML.

Para usar estos apuntes, debe ver tanto su visualización en el navegador, como su código desde un editor o desde el propio navegador (botón derecho del ratón -> menú contextual -> ver código fuente).

La estructura HTML, como el lenguaje de marcas, está basada en un conjunto de etiquetas que pueden contener atributos. De esta forma, la estructura que tendrán las etiquetas de forma genérica en su sintaxis será la siguiente:

<Nombredelaetiqueta nombredelatributo1="valor del atributo1" ... nombredelatributoN=valor del atributoN">Información contenida en la etiqueta </nombredelaetiqueta>

HTML es para la representación de información y estructura en la web. Pero necesita otros lenguajes para lo siguiente:

  • Visualización o estilo: se hace desde CSS.
  • Interacción con el usuario; con el modelo de atributos de evento, junto con lenguaje de programación JavaScript.

Ejemplos del modelo de eventos: cuando hacemos click en el botón, sale una ventana de alerta con mensaje.

Red Telefónica Conmutada (RTC)
La línea telefónica de toda la vida. Para acceder a Internet es necesario un módem.
Red Digital de Servicios Integrados (RDSI)
Una línea telefónica especial. Para acceder a Internet es necesario un módem RDSI.
Línea de Abonada Digital Asimétrica (ADSL)
Se basa en la conversión de una línea RTC a una línea de alta velocidad. Para acceder a Internet es necesario un módem ADSL.
Fibra óptica
Una línea de fibra óptica. Normalmente, la fibra óptica no llega hasta el usuario final, por lo que el término más apropiado es fibra híbrida coaxial.

Formularios

El lenguaje de HTML no solamente sirve para presentar información, también es usado para enviar datos a través de él al servidor o a algún servicio concreto.

Etiqueta <form>

Tiene unos atributos destacados en base a su funcionalidad:
Atributo method: Define la forma de comunicación con el protocolo por defecto usado (normalmente HTTP) y su valor por defecto es GET (envía los datos en la cabecera del paquete), POST (envía los datos en el cuerpo del paquete).
Atributo action: Vamos a poner el esquema y receptor de los datos enviados por el formulario; por defecto será el protocolo HTTP y la misma página donde estoy.
Atributo onsubmit: Es un atributo de evento, es decir, que podremos definir ejecuciones JavaScript. Para cortar el envío de datos, le daremos el valor "return false;".

Etiquetas <input> y <label>

Etiqueta <label>

Acompaña información a los campos de interacción, su atributo más importante es for que nos dice a qué campo de interacción se aplica mediante el id del campo.

Etiqueta <input>

Son campos de datos para el usuario.

Sus atributos más importantes son:

Atributo type: Define el dato que obtenemos del campo de datos, por defecto es text, pero podemos poner: button, checkbox, color, date, datetime-local, email, file, hidden, image, month, number, password, radio, range, reset, search, submit, tel, text, time, url, week.
Atributo name: Es importante para la comunicación, ya que el nombre del dato que se envía corresponde a lo que pongamos en este atributo.
Atributo value: Será el valor por defecto de un campo de entrada.
Atributo pattern: Significa que no dejará enviar los datos y los marcará como erróneos si no cumple el patrón o la expresión regular de este atributo. Tiene sentido en los campos de texto.
